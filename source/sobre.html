// Sobre o projeto

O que é

é o cenário escolhido para o Trabalho 03 de Cloud Computing: uma plataforma de podcasts e áudios que precisa de um ambiente Linux confiável, automatizado e fácil de operar. Toda a infraestrutura roda dentro de um container Docker baseado em Ubuntu, com o Apache servindo este site estático.

Como o ambiente é operado

Em vez de configurar tudo manualmente, a operação é feita por um conjunto de scripts Shell que cobrem as rotinas típicas de um time de DevOps:

  • Atualização do sistema e instalação do Apache (+ ffmpeg para áudio);
  • Criação da estrutura de pastas de episódios, áudios e transcrições;
  • Backup automatizado em .tar.gz com data e hora;
  • Deploy do site e publicação em /var/www/html;
  • Gerenciamento de processos e monitoramento de CPU, memória e disco;
  • Controle de usuários, grupos e permissões;
  • Geração de relatório operacional consolidado.

Tecnologias

Ubuntu Server, Docker e Docker Compose, Apache HTTP Server, Bash/Shell Script e ffmpeg para o processamento de áudio característico do tema.
